Definition

GOMS stands for Goals, Operators, Methods, and Selection Rules. It is a task analysis model that describes how users perform tasks by breaking them into structured steps.

Purpose of GOMS

The purpose of GOMS is to predict user performance and to analyze the efficiency of task procedures in an interface.

Components of GOMS

  • Goals: What the user wants to achieve
  • Operators: Basic actions performed by the user
  • Methods: Sequences of operators to achieve goals
  • Selection Rules: Rules for choosing between alternative methods

Example: Sending an Email

Goal: Send an email

  • Open email application
  • Type message
  • Click send

Advantages

  • Provides a structured description of tasks
  • Helps improve interface design
  • Identifies unnecessary steps

Limitations

  • Assumes skilled users
  • Does not model learning
  • Does not include emotional or social factors
